Instagram
AI DM Bot

A standing process that watches one Instagram thread, hands the full conversation to an LLM for every reply, and keeps a memory of what's already been said — built on instagrapi for session handling.

bot.py — session

$ python bot.py

MOD.01

LLM Replies

Every reply is generated from the entire thread, so the conversation stays coherent across many turns instead of answering each message in isolation.

MOD.02

Conversation Memory

Chat history is written to disk as it grows, so a restart picks up exactly where the conversation left off rather than starting cold.

MOD.03

Proxy Support

Both scripts accept a proxy URL, so traffic for login and messaging can be routed to match the account's usual network footprint.

Setup

Get It Running

Five phases, run in order. Each one assumes the previous phase finished cleanly — the bot won't start without a valid session, and it won't find the right thread without a configured target.

01/ 05

Clone the repository

Pull the project down and move into its directory before installing anything.

# clone the repo git clone https://github.com/izaart95-jpg/instaG-dm-bot.git cd instaG-dm-bot
02/ 05

Install dependencies

instagrapi is the unofficial Instagram client that handles login, session storage, sending and receiving DMs, and proxy configuration. requests is used for the LLM API calls.

# required pip install instagrapi # used for LLM calls pip install requests
03/ 05

Authenticate & store the session

Open login.py, drop in your account credentials, and run it once. It writes an authenticated session to session.json so the bot doesn't need your password on every run.

# login.py username = "your_email@example.com" password = "YourPassword123" # then run it python login.py

// tap or hover the blacked-out fields above — that's how your real credentials should look in any screenshot you share.

🔐 Logging in... ✅ Login successful! 💾 Session dumped to session.json
WARN

session.json is a live credential. Never commit it to a public repo — add it to .gitignore before your first commit.

04/ 05

Configure the bot

Open bot.py and update the values at the top of the file. At minimum, set TARGET_USER_ID to the account the bot should watch and reply to.

TARGET_USER_ID = 99123456789 # the IG user ID to chat with POLL_INTERVAL = 3 # seconds between message checks MAX_MSG_LENGTH = 900 # max chars per DM (IG limit)
05/ 05

Start listening

Loads the saved session, locates the target thread, and starts polling for new messages.

python bot.py
🔍 Looking for DM thread with user... ✅ Thread found: 340282366841710300000000000000000000 ℹ️ 3 existing messages marked as read. 🎧 Listening for new messages... ============================================================
OpSec Briefing

Proxy Configuration

Instagram watches for activity that doesn't look like a person on their phone. A login or a steady stream of DMs from a server IP — especially one in a different region from the account's usual logins — reads as automated and can get the account temporarily locked or asked to verify.

No proxy, datacenter IP
Flagged often
Residential proxy, matched region
Rarely flagged

Both login.py and bot.py have a commented-out proxy line — each script opens its own Client(), so the proxy needs to be set in both places.

# ─── login.py ─── cl = Client() cl.set_proxy("http://ip:port") # HTTP proxy # cl.set_proxy("socks5://ip:port") # SOCKS5 # cl.set_proxy("http://user:pass@ip:port") # authenticated cl.login(username, password)
# ─── bot.py ─── PROXY = "http://ip:port" cl = Client() cl.set_proxy(PROXY) # ... rest of setup
Field note 01

Use a residential or mobile proxy. Datacenter ranges are well known to Instagram's detection systems; residential IPs look like an ordinary home connection.

Field note 02

Match the account's home region. A proxy in the same country as the account's usual logins avoids "new location" warnings.

Field note 03

Set it in both files. login.py and bot.py each create their own client — a proxy in one doesn't carry over to the other.

Field note 04

Slow the poll interval down. 5–10 seconds between checks looks closer to a person glancing at their phone than a 1-second loop does.

Process

How It Runs

login.py

Authenticates with Instagram and writes the session to session.json. Run once, and again whenever the session expires.

python login.py

bot.py

The long-running process. Loads the saved session, finds the target thread, and polls it for new messages to answer.

python bot.py
Transmission Log — one message cycle
T+0.0s Poll fires — thread checked for anything new
T+0.0s New text message detected from the target user
T+0.1s Placeholder sent — "🤔 AI is thinking…"
T+0.2s Full conversation history posted to the LLM endpoint
T+1.4s Reply received — chunked automatically if over 900 chars
T+1.5s Reply sent and appended to memory for the next cycle
Reference

Configuration

All configurable values, set at the top of bot.py.

VariableDefaultDescription
TARGET_USER_ID 99123456789 Instagram user ID of the account the bot replies to
PROXY commented out HTTP / SOCKS5 proxy URL, e.g. "http://ip:port"
LLM_API_URL OpenAI-compatible endpoint Chat completions endpoint the bot sends conversation history to
LLM_MODEL provider-defined Model identifier passed to the LLM API
SYSTEM_PROMPT You are helpful AI... Defines the AI's personality and how it should respond
POLL_INTERVAL 3 Seconds between each check for new messages
MAX_MSG_LENGTH 900 Max characters per DM — longer replies are split automatically
